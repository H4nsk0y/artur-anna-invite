# Wedding Invitation

Мобильное свадебное приглашение с анимированным конвертом, музыкой, русской и армянской версиями и RSVP-анкетой.

## Запуск

```powershell
npm.cmd install
npm.cmd run dev
```

Сборка:

```powershell
npm.cmd run build
```

## Где менять данные

Имена, дата, адрес, расписание и оба перевода находятся в `src/content.ts`.

- Фотография: `public/couple.jpg`
- Музыка: `public/music.mp3`
- Дата для таймера: `event.isoDate`

Текущий армянский текст — черновой. Перед публикацией его нужно заменить вариантом заказчика.

## RSVP и Supabase

Без переменных окружения форма работает в демонстрационном режиме и сохраняет ответы только в `localStorage` конкретного браузера.

Для настоящего сбора ответов:

1. Создать проект Supabase.
2. Выполнить `supabase-schema.sql` в SQL Editor.
3. Скопировать `.env.example` в `.env.local`.
4. Заполнить `VITE_SUPABASE_URL` и `VITE_SUPABASE_PUBLISHABLE_KEY`.

Публичный ключ позволяет только добавлять ответы; чтение для гостей запрещено RLS-политикой. На первом этапе заказчик может смотреть ответы в Supabase Dashboard.

## Админ-панель

Адрес локально: `http://localhost:4173/#/admin`. После публикации админку можно открыть по `/#/admin`, а auth-callback Supabase возвращает пользователя на базовый адрес сайта.

Панель использует вход по одноразовой ссылке Supabase Auth и позволяет:

- видеть статистику и список ответов;
- искать гостей;
- выгружать CSV;
- удалять записи с подтверждением.

Для включения защищённого чтения и удаления:

1. Выполнить `admin-schema.sql` в Supabase SQL Editor.
2. В Supabase открыть **Authentication → URL Configuration** и добавить базовый адрес сайта в Redirect URLs.
3. На странице админки запросить magic link на email владельца.
4. После появления пользователя в **Authentication → Users** выполнить последнюю команду из `admin-schema.sql`, подставив email администратора.

Одна секретная ссылка не используется как защита: доступ проверяется серверными RLS-политиками через `auth.uid()`.

## GitHub Pages

Workflow уже находится в `.github/workflows/deploy.yml`. После создания репозитория и push в `main` нужно выбрать в настройках репозитория **Pages → Source → GitHub Actions**.

Для Supabase в GitHub добавить:

- Repository variable `SUPABASE_URL`
- Repository variable `SUPABASE_PUBLISHABLE_KEY`
